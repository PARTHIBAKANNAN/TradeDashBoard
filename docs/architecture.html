02 — Boundary

Security model

The architectural boundary is the whole point of the BFF: credentials and the broker connection are quarantined server-side.

✕Never on the frontend

API keys · Client ID · Secret Key · TOTP seed · trading PIN · access token · the raw broker websocket.

✓Backend-only

Loaded from .env via python-dotenv; the daily token is cached to .token_cache.json (git-ignored).

🛡️

Corporate TLS. On a network with SSL inspection, Python's bundled CA store rejects the broker's cert. pip-system-certs routes TLS through the Windows certificate store (which holds the corporate root CA), fixing requests, the FYERS SDK, and the websocket at once.

03 — Authentication

Login method & flow

Fyers access tokens expire every 24 hours. Rather than a manual OAuth redirect, the backend drives Fyers' programmatic login using your own credentials + a TOTP seed — the standard headless algo pattern. A newly created app requires a one-time browser consent; every day after is fully automatic.

The daily automated chain

1

Send login OTP challenge

POST /vagator/v2/send_login_otp_v2 — the base64 login id opens a TOTP session.

# api-t2.fyers.in
{ "fy_id": base64(FY_ID), "app_id": "2" }  → request_key  (sub: totp_login)
2

Verify TOTP

POST /vagator/v2/verify_otp — code from pyotp.TOTP(seed).now(). Generated boundary-safe (avoids the tail of the 30s window) with a one-shot retry.

{ "request_key": …, "otp": TOTP() }  → request_key  (sub: verify_pin)
3

Verify PIN → vagator token

POST /vagator/v2/verify_pin_v2 — the _v2 variant is required; the base64 PIN yields a short-lived login access token.

{ "request_key": …, "identity_type": "pin", "identifier": base64(PIN) }
    → data.access_token  (the "vagator" bearer)
4

Obtain the OAuth auth code

POST /api/v3/token with the vagator bearer. For an authorized app this returns a redirect URL carrying the real auth_code (sub: auth_code).

# api-t1.fyers.in
Bearer <vagator_token>
{ "fyers_id", "app_id", "redirect_uri", "appType",
  "response_type": "code", "create_cookie": true }
    → Url?auth_code=…&state=dashboard
5

Exchange for the API access token

SessionModel.generate_token() POSTs to /validate-authcode with appIdHash = SHA256("client_id:secret") + the auth code.

session.set_token(auth_code)
token = session.generate_token()["access_token"]  # valid until end of day
6

Cache & use

Token is date-stamped into .token_cache.json (survives restarts today), then powers the REST FyersModel and the FyersDataSocket feed (CLIENT_ID:token).

One-time consent for a new app

A freshly created app is not yet authorized against your account, so step 4 returns a pre-consent data.auth token (sub: access_token) with no auth_code — which /validate-authcode rejects with -437. Running manual_auth.py once (browser login → Allow → paste the redirected URL) authorizes the app permanently. After that, the chain above runs unattended.

Token lifecycle

⏰08:45 IST cron

APScheduler force-refreshes a fresh token before the open — no manual step.

💾Same-day cache

Restarts reuse the cached token; a stale (previous-day) cache triggers re-login.

🔁Graceful fallback

If login fails, the server stays up and serves the last snapshot instead of crashing.

04 — Data plane

05 — Engines

Mathematical calculations

Every tick recomputes four deterministic engines in memory before the next broadcast. All functions are pure and unit-tested.

Engine A

Intraday Relative Strength vs NIFTY 50

IRSi,t = Δ% Stocki,t − Δ% Indext

Each side is % change vs its own previous close. The sign classifies the row:

IRS > 0 · Outperform · #22c55e IRS < 0 · Underperform · #ef4444
Engine D

Day Range Position (%)

LTPt − TLTH − TL × 100

Where the price sits inside today's low→high band. Feeds the slider filter directly; a zero-span guard returns 0 to avoid division errors.

Engine B

30-min Opening Range Breakout (C1–C4)

UpperCn = max(P)  ·  LowerCn = min(P)  over t ∈ Cn

Candles: C1 09:15–09:45 · C2 09:45–10:15 · C3 10:15–10:45 · C4 10:45–11:15. Once a candle completes, a breach of its bound triggers a signal; the newest completed candle wins, and repeated same-state ticks don't re-fire.

LTP > Upper → Bull • Cn LTP < Lower → Bear • Cn
Engine C

Normalized Dual-Range Mapper

x(P) = P − minmax − min × 100

With min = min(YL, TL) and max = max(YH, TH), both sessions map onto one 0–100% scale so yesterday's grey bar, today's blue bar, and the LTP tick share coordinates on a single canvas element.

✓

Verified. The engine suite passes its unit tests — python -m tests.test_calculations (pct-change, IRS, day-range, dual-range map, ORB breakout & precedence).

07 — Operations

Daily lifecycle

APScheduler on the IST timezone; if the process boots mid-session it jumps straight to the correct state.

☀

08:45 — Daily token

Force-refresh a fresh access token via the automated TOTP chain (Mon–Fri).

▶

09:15 — Market open

Backfill ranges + ORB, flip market_open = true, launch the websocket thread.

≈

09:15–15:30 — Live

Ticks stream in; SSE broadcasts the computed state once per second.

■

15:30 — Standby

Close the websocket to conserve bandwidth; UI shows Closed with the final snapshot.

🧪

Dev override. FORCE_MARKET_OPEN=true in .env runs the engine regardless of the clock to exercise the full pipeline outside market hours.
